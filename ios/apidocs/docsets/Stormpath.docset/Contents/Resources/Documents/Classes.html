The following classes are available globally.

  • Stormpath

    Stormpath represents the state of the application’s connection to the Stormpath Client API. It allows you to connect to the Stormpath Client API, register, login, and stores the current account’s access and refresh tokens securely. All callbacks to the application are handled on the main thread.

  • StormpathConfiguration

    StormpathConfiguration is the class that manages the configuration for Stormpath, its endpoints, and API providers. It auto-loads from the configuration in Info.plist, or uses defaults that connect to a server on http://localhost:3000, the default in many Stormpath integrations. You can modify its properties directly.

    Note

    The endpoints refer to the endpoints in the Stormpath Framework Spec. Use leading slashes to specify the endpoints.

  • StormpathError

    StormpathError represents an error that can be passed back by a Stormpath API after a network response. It typically contains a HTTP status code, along with a localizedDescription of the specific error as passed back by the Framework Integration API. It also has two special error codes:

    0 - Internal SDK Error (which should be reported to Stormpath as a bug) 1 - Unrecognized API Response (which means the Framework integration may not support this version of Stormpath)
